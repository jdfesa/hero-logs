# Architecture Decision Records

This folder stores important technical and product decisions.

Use an ADR when a decision:

- Changes architecture.
- Adds a major dependency.
- Introduces a sensitive permission.
- Changes local data storage.
- Changes scoring rules.
- Creates a long-term product constraint.

## Format

Each ADR should include:

- Status.
- Context.
- Decision.
- Consequences.
- Alternatives considered.
- References when useful.

## Naming

Use sequential filenames:

```text
0001-native-android-kotlin-compose.md
0002-local-first-data-storage.md
0003-background-location-strategy.md
```
